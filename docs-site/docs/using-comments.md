# Using Comments

How to leave and manage review comments on a Docusaurus site that has this plugin installed.

## Leaving a Comment

1. Select any text on the page
2. A **comment button** appears next to your selection
3. Click it to open the comment form
4. Choose a comment type and write your comment
5. Submit — the comment is saved immediately

## Comment Types

### Question

Use when something is unclear or you need more information.

> "What does the `contextDirs` option do?"

### Suggestion

Use when you have an idea to improve the documentation.

> "Consider adding a diagram to visualize the data flow."

### Issue

Use when you've found an error or a problem in the documentation.

> "This code snippet is missing a closing bracket."

## Replying to Comments

Click on any comment to expand it, then type in the reply field and submit.

## Resolving Comments

Once a comment has been addressed, click **Resolve** to mark it as resolved. Resolved comments are hidden by default but can be shown again with the filter toggle.

## Tips

- **Be specific.** "This is confusing" is less useful than "The relationship between X and Y is unclear."
- **One topic per comment.** Create separate comments for separate issues.
- **Use the right type.** Questions get answers; suggestions get evaluated; issues get fixed.
